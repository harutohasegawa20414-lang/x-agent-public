# ダッシュボード「このサイトにアクセスできません」（ERR_CONNECTION_REFUSED）原因と解決順

`localhost:5001` にアクセスしたときに「接続が拒否されました」となる事象について、ログ（`frontend.log`・`backend.log`）と設定を確認した結果をまとめます。**改善は行わず、原因の洗い出しと解決すべき順番の整理のみ**行います。

---

## 1. 原因の洗い出し

### 原因A: ダッシュボード（Vite）の開発サーバーが起動していない【直接原因】

- **状況**: ブラウザで `http://localhost:5001` にアクセスすると、**ERR_CONNECTION_REFUSED**（接続が拒否されました）となる。
- **ログの根拈**:
  - `frontend.log` では、過去に `VITE v6.4.1 ready` で `Local: http://localhost:5001/` が表示されており、**その時点では** Vite が 5001 で待ち受けていた。
  - いま接続できないということは、**現在は 5001 番で待ち受けているプロセスがない**状態と考えられる。
- **設定の根拈**: `dashboard/vite.config.js` で `server.port: 5001` となっており、ダッシュボードは 5001 で起動する想定。
- **結論**: **5001 でリッスンする Vite の開発サーバーが起動していない**ことが、接続拒否の直接の原因である。

---

### 原因B: フロントエンドの JSX 構文エラー（SchedulerPanel.jsx）

- **状況**: `frontend.log` に以下のエラーが記録されている。
  - **Adjacent JSX elements must be wrapped in an enclosing tag**（`SchedulerPanel.jsx` 320 行付近）
  - **Unexpected token, expected ","**（同 372 行付近）
  - **Duplicate key "display" in object literal**（複数箇所の `style` で `display` が二重）
- **影響**: サーバー自体は起動済みのログがあるため、**初回起動はできている**。これらのエラーは主に HMR（ホット更新）時やビルド時に問題になり、**ダッシュボードが開けない直接原因ではない**。ただし、起動後に該当コンポーネントを開いた際に表示崩れやエラーになる可能性はある。
- **結論**: 接続拒否の**直接原因ではない**が、**ダッシュボードを開いたあとの安定性・表示**の観点では解消した方がよい。

---

### 原因C: バックエンドのスケジューラ二重起動（SchedulerAlreadyRunningError）

- **状況**: `backend.log` に `SchedulerAlreadyRunningError: Scheduler is already running` が記録されている。`api/main.py` の `update_scheduler_config` 内で `await scheduler_instance.start()` を呼んでいるが、スケジューラは既に lifespan で起動済みのため、設定更新時に再度 `start()` を呼ぶとエラーになる。
- **影響**: ダッシュボードからスケジューラ設定を保存した際に **500 Internal Server Error** が返り、設定が保存できない場合がある。**localhost:5001 に接続できない原因ではない**。
- **結論**: 接続拒否の**直接原因ではない**。**API の設定保存**を安定させるために解消すべき。

---

### 原因D: X（Twitter）API の 403 Forbidden（投稿権限）

- **状況**: `backend.log` に `Error posting to X: 403 Forbidden`、`You are not permitted to perform this action.` および Free アカウントの場合は App Permissions を Read and Write にする旨のメッセージが記録されている。
- **影響**: ダッシュボード表示や 5001 への接続には無関係。**投稿実行時**に失敗する事象。
- **結論**: 接続拒否の**直接原因ではない**。**投稿機能**を使う場合に解消すべき。

---

## 2. 解決すべき順番

「まずダッシュボードにアクセスできるようにする」ことを最優先にし、そのあとでダッシュボード利用時の不具合・投稿まわりを並べています。

| 順番 | 原因 | 理由 |
|------|------|------|
| **1** | **原因A: ダッシュボード（Vite）の開発サーバーが起動していない** | これが解消されないと `localhost:5001` に一切つながらない。**最優先で対応すべき**。 |
| **2** | **原因B: SchedulerPanel.jsx の JSX 構文エラー** | ダッシュボードは開けるようになったあと、スケジューラまわりの表示・操作でエラーや不具合が出る可能性があるため、**2 番目**に解消するのがよい。 |
| **3** | **原因C: スケジューラの二重起動（SchedulerAlreadyRunningError）** | ダッシュボードからスケジューラ設定を保存する際に 500 が出る問題。**API の安定性**の観点で **3 番目**。 |
| **4** | **原因D: X API 403（投稿権限）** | ダッシュボード表示・API の起動とは独立した、**投稿機能**だけの問題。**4 番目**。 |

---

## 3. 補足（ログ・設定からの整理）

- **フロント**: `dashboard/package.json` の `"dev": "vite"` で `dashboard` ディレクトリから `npm run dev` を実行すると、`vite.config.js` に従い **5001** で待ち受ける。
- **バックエンド**: `api/main.py` で `uvicorn.run(app, host="0.0.0.0", port=5002)` となっており、API は **5002** で待ち受け。ダッシュボードは `/api` を 5002 にプロキシする設定。
- **接続拒否の理由**: 5001 でリッスンするプロセス（Vite）が**いま動いていない**ため、ブラウザの接続が拒否されている。

---

## 4. まとめ

- **「このサイトにアクセスできません」の直接原因**: **原因A（ダッシュボードの開発サーバーが起動していない）**。
- **解決の順番**:  
  1. 原因A（Vite を 5001 で起動する）  
  2. 原因B（SchedulerPanel.jsx の JSX 修正）  
  3. 原因C（スケジューラ二重起動の防止）  
  4. 原因D（X API の権限設定）

以上、改善は行わず原因の洗い出しと解決順の整理のみとしています。
