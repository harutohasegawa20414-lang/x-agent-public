# Cloud Code サインイン後の次のステップ

Cloud Code のパネルが開き、サインインとプロジェクト選択が完了したあとにできることです。

---

## 今すぐやること（必須ではない）

**何も必須の操作はありません。**  
Cloud Code は「使いたいときに使う」ツールです。左側の **KUBERNETES** や **CLOUD RUN** などのメニューは、その機能を使うための入口です。

---

## 左側メニューの意味と使い方

| メニュー | 用途 |
|---------|------|
| **KUBERNETES** | GKE（Google Kubernetes Engine）クラスターの作成・接続・管理。コンテナを複数台で動かすときに使う。 |
| **CLOUD RUN** | サーバーレスでコンテナやアプリをデプロイ・管理。このプロジェクトの API（FastAPI）を公開するのに向いている。 |
| **CLOUD FUNCTIONS** | イベントや HTTP で動く小さな関数をデプロイ。軽い処理向け。 |
| **CLOUD BUILD** | ビルド・デプロイの自動化（CI/CD）。 |
| **その他** | プロジェクト内のリソース一覧、ヘルプ・フィードバックなど。 |

---

## このプロジェクト（Xエージェント）でやりたいことに合わせた選択

### 1. 「とりあえず設定完了でOK」の場合

- **何もしなくて大丈夫です。**
- 必要になったときに Cloud Code パネルから CLOUD RUN や KUBERNETES を開けばよいです。

---

### 2. 「API をインターネットに公開したい」場合（おすすめ）

- **CLOUD RUN** を使うと、このプロジェクトの **FastAPI（`api/main.py`）** を簡単にデプロイできます。

**手順の流れ：**

1. Cloud Code パネルで **CLOUD RUN** をクリック
2. **「Deploy to Cloud Run」** や **「サービスをデプロイ」** などのオプションを選ぶ
3. 案内に従って、デプロイするフォルダ（例: プロジェクトルートや `api` があるフォルダ）や Dockerfile の有無を選択
4. 初回は **Dockerfile の作成** や **リージョン選択** を求められることがあるので、画面の指示に従う

※ まだ Dockerfile がない場合は、Cloud Code が「Dockerfile を生成する」提案をしてくれることがあります。

---

### 3. 「Kubernetes（GKE）で動かしたい」場合

- **KUBERNETES** をクリック
2. **「Create Cluster」** や **「クラスターを作成」** を選ぶと、GKE クラスターの作成ウィザードが開く
3. クラスター作成後、**「Connect」** で Cursor から接続できる

※ GKE は有料リソースなので、料金を確認してから作成してください。

---

### 4. 「まずは中身を確認したい」場合

- **CLOUD RUN** をクリック → 既存の Cloud Run サービスがあれば一覧表示される
- **KUBERNETES** をクリック → 既存のクラスターがあれば一覧表示される
- 各メニューを開いて、現在のプロジェクトに何があるか確認するだけでもOKです

---

## まとめ：次に何をすればいいか

| やりたいこと | やること |
|--------------|----------|
| 特に何もしたくない | 何もしなくてOK。必要になったらパネルを開く。 |
| API を公開したい | **CLOUD RUN** → デプロイの案内に従う。 |
| Kubernetes を使いたい | **KUBERNETES** → クラスター作成 or 既存クラスターに接続。 |
| 様子を見たい | **CLOUD RUN** や **KUBERNETES** を開いて一覧を確認。 |

---

## 注意

- **Cloud Run** や **GKE** を使うと、Google Cloud の利用料金が発生する場合があります。  
  初回は無料枠・クレジットを確認してからデプロイやクラスター作成を行うと安全です。
- このプロジェクトは現状、ローカルの API とダッシュボードで完結しているので、**Cloud Code の設定が終わった時点で「次にやること」は完了**と考えて問題ありません。
